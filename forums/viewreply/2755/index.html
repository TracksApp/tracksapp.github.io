For the benefit of visitors of this thread I will explain how I finally did it.
It was not as smooth as I expected to put Tracks 2.0 RC2 (37) on Hostgator, but I think these issues are more related with Hostgator rather than with Tracks.
1) Some required gems were not installed on Hostgator by default. But - there is a really simple tool (Ruby gems installer) to install gems from Hostgator’s cPanel. I still couldn’t install all required gems due to permission issues, but admins in Hostagotr kindly did it for me. Required gems:

highline
RedCloth (asked to install for me)
soap4r
datanoise-actionwebservice (asked to install for me)
sanitize (asked to install for me)
nokogiri (asked to install for me)
rack
will_paginate
has_many_polymorphs

to install using SSH, use:
gem install <gem name>

In cPanel just enter the <gem name> in installation field and press INSTALL

2) I could run (start) ruby application for Tracks 1.7.3 from cPanel (there’s a little “RUN” button), but 2.0RC2 would not run. I started the server with “script/server -e production” in putty (SSH port in Hostgator is 2222) and it started successfully on port 3000. Hostgator runs ruby apps on 1200x ports. In cPannel there is also a simple tool to redirect a subdomain/domain or a directory to application (not really necessary if you know the port your app is running, but if you want an easy access, redirect is a nice feature). Anyway there is a workaround… Create the redirect with cPanels rewrite tool (under Ruby on Rails), then open File Manager and navigate to the subdomain/domain/directory where the redirect is located and edit the .htaccess file. Change the port 1200x to 3000. That’s it.

RewriteEngine on

RewriteCond %{HTTP_HOST} ^sub.domain.com$ [OR]
RewriteCond %{HTTP_HOST} ^www.sub.domain.com$
RewriteRule ^(.*)$ "http://127.0.0.1:3000/$1" [P,L]

Hey guys,

Has anyone tried to run the 2.1devel trunk behind Apache yet?

When I brought down the latest devel tracks from git, I got it working in mongrel, but it wouldn’t work with my Apache setup that used to work just fine.

The error that was coming up was the Passenger module complaining that “bundle install” didn’t check out the actionwebservice from git as specified in the Gemfile:

git://github.com/dejan/actionwebservice.git is not checked out. Please run `bundle install` (Bundler::GitError) 

But that wasn’t true, I confirmed that the bundler did properly download actionwebservice gem.

After many hours googling like a madman, I found a workaround in this forum post: http://groups.google.com/group/snorby/browse_thread/thread/56da39bf2f502848

Running these bundle commands in the root of tracks solved the problem for me:

bundle pack
bundle install --path vendor/cache 

I believe this is a bug in passenger, as that’s what is alluded to in this SE question: http://stackoverflow.com/questions/5007773/installing-gems-with-bundler-big-problem.

It might be a good idea to put this caveat in the wiki for installation.

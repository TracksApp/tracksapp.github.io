I have tried from your test account and I found that all delete actions fail (i.e. delete action, delete project, delete context). When I look using Firebug I see that the webbrowser does the correct request (HTTP-DELETE over SSL)

Since the logs of tracks and passenger do not show anything and the browser is doing the right thing, I think this could be a problem with your NGINX setup, i.e. the request is not passed on to tracks via passenger.

Do you have any rewrite rules that make HTTP DELETE fail? Are there nginx logs (like error_log for apache)?

Hello all,
I have investigated further the issue. I tried to access directly the Tracks instance and then the Delete function just worked.
In fact I realized that the default for Pound was to reject DELETE and HEAD HTTP commands. I had to add the “xHTTP 1” option to the Pound config file to accept the requests and get the Tracks application working.

Problem solved - thank you for your help investigating in the problem !

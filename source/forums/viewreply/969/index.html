Right way to unhide an orphan hidden action?
 
marcianx
Posted: 30 December 2008 03:53 PM   [ Ignore ]  
Newbie
Total Posts:  9
Joined  2008-12-30

I created an action in a hidden project. Then I realized that the action no longer belonged in that project. Then I cleared its project field, and strange stuff started happening.

The item does not show up under its context on the main page. Searching for the item brings it up, but there is no edit button. It was fortunate that I had added a tag to it. When I clicked on the tag in the search result, it showed up that item again, and this time with an edit button. It comes under “Hidden actions tagged with <tagname>.” So then, I cumulatively did the following actions.

1. I edited the item’s “Show From” and “Due” fields to today,  but it’s still hidden in other views.
2. I put it under a visible Project, but it’s still hidden from the context view and the main page, and the project’s # actions counter doesn’t go up.
3. I put it under a new temporary project (its # actions counter didn’t go up). But then, I made the project hidden, and is # actions counter DID go up. Then, I made the Project visible, and the action also became visible.

So, is there a better way to do this? Here are a couple of suggestions for improvements.

1. It seems that visibility is a per-action property, but it’s settable only per project. I would probably just make it a per-project property.
2. Search results should have an edit icon.

this is a cool scenario! The problem is that we keep track of the hidden state on projects and on todos. When you removed the project from the todo, the todo remained hidden. There is indeed no ‘normal’ way to recover a hidden todo. Your #3 is I think the only way possible.

This is a bug. We should clear the hidden state of a todo when its orphaned…

Could you create a ticket for this? http://www.assembla.com/spaces/tracks-tickets/tickets
